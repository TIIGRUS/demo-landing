# Service Worker

Офлайн-режим на основе ручного Service Worker без сторонних зависимостей. Кэшируются локальные ассеты и главная страница; внешние сервисы (аналитика, видео) не кэшируются.

## Особенности

- 📦 **Precache** — при установке сразу кэшируются главная страница, манифест и offline-страница
- 🔄 **Runtime cache** — локальные ассеты кэшируются по мере первого посещения
- 📡 **Network-first для навигации** — всегда свежая страница при наличии сети
- 🚫 **Без кэша third-party** — Yandex Metrica, YouTube, Vimeo идут только через сеть
- 🌐 **Совместимость с GitHub Pages** — пути строятся через `self.registration.scope`

---

## Файлы

| Файл                  | Назначение                                      |
| --------------------- | ----------------------------------------------- |
| `public/sw.js`        | Логика Service Worker: install, activate, fetch |
| `public/offline.html` | Fallback-страница при отсутствии сети           |
| `src/main.ts`         | Регистрация SW (только в production)            |

---

## Регистрация

В `src/main.ts` в конце файла:

```typescript
if ('serviceWorker' in navigator && import.meta.env.PROD) {
    window.addEventListener('load', () => {
        navigator.serviceWorker
            .register(`${import.meta.env.BASE_URL}sw.js`)
            .catch((error) => console.error('Service Worker registration failed:', error));
    });
}
```

`BASE_URL` берётся из Vite-конфига автоматически — совместимо с `base: './'` и GitHub Pages.

## Кэш-стратегии

| Тип запроса                                           | Стратегия                     | Поведение в офлайне                                  |
| ----------------------------------------------------- | ----------------------------- | ---------------------------------------------------- |
| Навигация (`navigate`)                                | Network-first                 | Возвращает закэшированную главную или `offline.html` |
| Локальные ассеты (img, css, js, fonts)                | Cache-first с runtime caching | Отдаёт из кэша; при отсутствии — сеть + кэшируется   |
| Внешние домены (mc.yandex.ru, youtube.com, vimeo.com) | Network-only                  | Не кэшируются; при офлайне ошибка изолирована        |
| Не-GET запросы                                        | Пропуск                       | SW не вмешивается                                    |

---

## Важно

**Ресурс попадает в runtime-кэш только после первого онлайн-запроса.**
Это касается lazy-loaded изображений: для их офлайн-доступности нужно хотя бы раз прокрутить страницу до конца при включённой сети.

**При каждом новом билде:**

1. Новый SW при активации удаляет старый кэш (`activate`).
2. CSS/JS нужно заново «прогреть» — открыть страницу online и прокрутить до конца.
3. Только после этого переключать Offline для проверки.

**Версионирование кэша:**
При изменении логики SW обязательно обновить значение `CACHE_NAME` (например, `landing-cache-v2`). Иначе `activate` не удалит старый кэш.

---

## Проверка после деплоя

1. Открой прод URL онлайн, прокрути страницу до конца.
2. DevTools → Application → Service Workers: статус `activated and running`.
3. DevTools → Application → Cache Storage: проверь наличие `landing-cache-v1`.
4. DevTools → Network → Offline: обнови страницу — сайт должен открыться.
5. Убедись что в Console нет необработанных ошибок (`503 Offline` для некэшированных ресурсов допустимо).
